# Parallel Mergesort Client-Server Application

Това е Java проект, реализиращ клиент-сървърна архитектура за паралелно сортиране на данни. Приложението използван **Java Sockets**, **Object Serialization** и **Fork/Join Framework** за многонишково програмиране.

## Функционалности

1.  **Multithreaded Server:** Сървърът може да обслужва множество клиенти едновременно.
2.  **Parallel Merge Sort:** Използва `RecursiveAction` и `ForkJoinPool` за разпределяне на сортирането между всички налични процесорни ядра.
3.  **Сравнителен анализ:** Клиентът автоматично измерва и сравнява времето за изпълнение при една нишка (sequential) спрямо много нишки (parallel).
4.  **TCP Комуникация:** Обмен на данни чрез сериализирани Java обекти (`SortRequest`, `SortResponse`).
5.  **Thread Safety:** Напълно thread-safe архитектура без споделено състояние (shared mutable state).

## Технически Изисквания

* **Java JDK:** версия 24 или по-нова.
* **Apache Maven:** за управление на зависимостите и компилация.

## Архитектура на системата
Приложението следва класическа Клиент-Сървър архитектура, комуникираща чрез TCP сокети.
**Сървър**
Сървърът е проектиран да бъде мащабируем и thread-safe. Основната нишка слуша на порт 5555. При всяко свързване на нов клиент, сървърът отделя нова нишка (ClientHandler), която поема комуникацията. Това гарантира, че сървърът не блокира и може да обслужва N клиента едновременно. За същинското сортиране се използва ForkJoinPool, който разпределя задачата между наличните нишки.

**Клиент**
Клиентът е конзолно приложение, което:
1. Генерира тестови данни (масиви от числа или координати).
2. Изпраща заявка към сървъра.
3. Получава сортирания масив и статистиката за времето на изпълнение.

**Протокол за комуникация**
Комуникацията се осъществява чрез сериализация на Java обекти:
SortRequest съдържа данните за сортиране (int[] или Point[]) и желания брой нишки.
SortResponse съдържа сортирания масив и времето за изпълнение в наносекунди.

## Стъпки за стартиране
**Компилиране**: Отворете терминал в главната папка (където е pom.xml) и изпълнете:
mvn clean install

**Стартиране на Сървъра**: Изпълнете:
java -cp target/classes server.Server

**Стартиране на Клиента**: Отворете нов терминал и изпълнете:
java -cp target/classes client.Client
